# Правила стилизации и форматирования

## Tailwind CSS

### Рекомендации

1. Предпочитать Tailwind utility классы кастомному CSS
2. Использовать переменные темы для цветов
3. Избегать инлайн стилей, использовать классы
4. Следовать Stylelint правилам при написании CSS
5. Использовать `tailwind-merge` для объединения классов с переопределениями

## Форматирование кода

### Отступы

- Использовать 2 пробела для отступов
- Настройки ESLint автоматически применяются через `npm run lint:fix`

### Точки с запятой

- Использовать точки с запятой (настройка ESLint)
- Автоматически применяется через линтер

### Длина строк

- Стараться не превышать 100-120 символов в строке
- Переносить длинные выражения на несколько строк
